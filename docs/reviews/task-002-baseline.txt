usage: xlsx_add_comment.py [-h] [--cell REF] [--text MSG] [--author NAME]
                           [--initials INI] [--batch FILE]
                           [--default-author NAME] [--default-threaded]
                           [--threaded] [--no-threaded]
                           [--allow-merged-target] [--date ISO]
                           [--json-errors]
                           input output

Insert a Microsoft Excel comment into a target cell of a .xlsx workbook.

positional arguments:
  input                 Source .xlsx/.xlsm.
  output                Destination .xlsx/.xlsm (must differ from input).

options:
  -h, --help            show this help message and exit
  --cell REF            Target cell. Forms: A5, Sheet2!B5, 'Q1 2026'!A1,
                        'Bob''s Sheet'!A1.
  --text MSG            Comment body (plain text). Empty or whitespace-only →
                        exit 2 EmptyCommentBody.
  --author NAME         Display name. Required when --cell.
  --initials INI        Override initials (default: first letter of each
                        whitespace-token in --author).
  --batch FILE          JSON file (or - for stdin). Auto-detects flat-array vs
                        xlsx-7 envelope. 8 MiB cap.
  --default-author NAME
                        Required when --batch is xlsx-7 envelope shape;
                        ignored otherwise.
  --default-threaded    Default `threaded` for envelope-shape rows.
  --threaded            Force threaded write (writes BOTH legacy stub +
                        threadedComment + personList per Q7).
  --no-threaded         Force legacy-only write (no threaded part, no
                        personList).
  --allow-merged-target
                        Redirect comment to anchor of merged range instead of
                        failing fast (R6.b).
  --date ISO            Override timestamp on <threadedComment dT>. Default:
                        datetime.now(UTC).
  --json-errors         Emit failures as a single line of JSON on stderr
                        (machine-readable: {error, code, type?, details?}).
