## Notes
- For `huge-100k-rows.xlsx` data shape: 100K rows × 10 columns; data deliberately uniform (so cache replay fires meaningfully) but with 5 % noise (out-of-range hours, blank cells) to give the rules something to find. Final size ~10–15 MB.

- The `test_xlsx7_to_xlsx6_pipe_*` tests use `subprocess.run` with `stdin=PIPE` for the pipe leg. The xlsx-6 binary path is `skills/xlsx/scripts/xlsx_add_comment.py`. xlsx-6 has been merged; this is exercising real installed code, not a mock.
- The peak-RSS measurement via `resource.getrusage` returns kilobytes on Linux, **bytes** on macOS — the comparison `<= 500*1024*1024` works on Linux directly (500 MB in KB == 500 * 1024); on macOS the figure is in bytes already. Use a platform check or just leave the larger threshold and document.
