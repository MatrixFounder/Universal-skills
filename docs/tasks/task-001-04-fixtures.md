### Concrete README content

```markdown
# Goldens — agent-output-only protocol

These files are **agent-output-only** (R9.d / TASK §5 Assumptions).

## Why
Excel 365 may silently mutate legacy comments → threaded on save (per
backlog xlsx-6 honest scope). If a golden is opened and saved in Excel,
the next regression run will diff against the mutated file, producing
false positives.

## Rules
1. **DO NOT open these files in Excel** (or Numbers, or any consumer
   that might rewrite OOXML on save). LibreOffice has the same risk;
   if you must spot-check, open a COPY.
2. To regenerate, re-run `xlsx_add_comment.py` with the documented
   fixture inputs and `--date 2026-01-01T00:00:00Z` for determinism.
3. Inputs (`golden/inputs/`) ARE allowed to be Excel-authored — they
   are immutable for the lifetime of the test. Provenance:
   - `with_legacy.xlsx`, `macro.xlsm` — authored in Excel 365 (Office
     Insider build 16.85), preserved verbatim. Do NOT regenerate
     unless the OOXML semantics under test change.
   - `clean.xlsx`, `multi_sheet.xlsx`, `hidden_first.xlsx`,
     `merged.xlsx` — generated by openpyxl; regeneration script:
     `python3 tests/regenerate_synthetic_inputs.py`.
   - `encrypted.xlsx` — `office_passwd.py --encrypt password123` over
     `clean.xlsx`.
4. Outputs (`golden/outputs/`) are diffed via canonical XML compare
   (`lxml.etree.tostring(method='c14n')`, with `<threadedComment id>`
   and unpinned `dT` masked) — see task 2.10.
```
