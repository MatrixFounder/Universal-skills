## 1. Что из прошлых рекомендаций УЖЕ работает (проверено на #01) ✅

Обновление закрыло ключевые пункты прошлой постановки — подтверждаю на реальных ссылках:

- **Типизированная диагностика fetch-фейла (бывш. R-2).** `_fetch_kind()` классифицирует
  статус в `details.kind` (`forbidden` / `auth_required` / `not_found` / `rate_limited` /

  - `arxiv.org/html/2204.00251` → чистый `{"error":"...HTTP 404","code":10,"type":"FetchFailed",

- **403 → эскалация на браузерный UA + retry/backoff + 429/`Retry-After` + `--rate-limit`/`--retries`.**
  Честный UA по умолчанию, браузерный — только как одна попытка при 403. Это и дало главный выигрыш ниже.
- **`--engine jina`** (Jina Reader, серверный JS-рендер без локального браузера) — добавлен.
- **Точность даты (бывш. R-4).** `_jsonld_date` / `_arxiv_date_from_url` / `_structured_date` —
  именно тот arxiv-баг с датой из прошлого фидбэка.
- **X/Twitter теперь берётся — БОЛЬШОЙ выигрыш.** Раньше треды были недоступны; теперь:
  - Justin Bons «коллапс BTC» → **26 КБ** (вся аргументация треда),
  - Vitalik «рынки предсказаний» → 5 КБ, zeror1sk → 5 КБ.

## 3. Новые дефекты (что нужно доработать)

### Дефект №1 — Wikipedia: НЕМОЙ пустой вывод (exit 0, тело пустое). **Серьёзность: ВЫСОКАЯ**
Все 3 wiki-URL (`en.wikipedia.org/wiki/...`, `ru.wikipedia.org/wiki/...`) дают **exit 0**,
но в выводе только YAML-фронтматтер (~220 байт), **тела нет** — и в whole-page (`--no-reader`),
и в reader-режиме. Это худший класс отказа: выглядит как успех, молча теряет контент.

Репро (любой из трёх):
```
python3 scripts/html2md.py "https://en.wikipedia.org/wiki/Value_averaging" --stdout --no-reader
# exit 0; на stdout — только фронтматтер, тело пустое
python3 scripts/html2md.py "https://ru.wikipedia.org/wiki/Long-Term_Capital_Management" /tmp/wk
# /tmp/wk/*.md и *.reader.md = 220 байт каждый (пусто)
```
Похоже, trafilatura/reader не извлекает основной контент из текущей HTML-разметки Wikipedia.

**Рабочий обход (использован в импорте), конвертация остаётся детерминированной (скилл):**
```
curl -sL -A "Mozilla/5.0" "https://en.wikipedia.org/api/rest_v1/page/html/Value_averaging" -o va.html
python3 scripts/html2md.py va.html --stdout --no-reader      # → 17–78 КБ чистой статьи
```
REST-эндпоинт `/<lang>.wikipedia.org/api/rest_v1/page/html/<Title>` отдаёт чистый article-HTML,
который скилл конвертит нормально.

### Дефект №2 — утечка отладочной строки в stderr (неверный префикс). **Серьёзность: НИЗКАЯ**
При reader-фетче в stderr попадает обрывок: `html2pdf: reader-mode root via ` (строка обрезана,
и префикс — `html2pdf`, хотя это html2md). Косметика, но сбивает с толку и шумит в `--json-errors`-потоке.

### Дефект №3 — arXiv `/html/<id>` 404 для старых статей. **Серьёзность: НИЗКАЯ (nice-to-have)**
`arxiv.org/html/2204.00251` (статья 2022 г.) → HTTP 404, т.к. HTML-версии нет (есть только PDF).
Скилл сообщает корректно (это плюс типизированной диагностики). Удобство: при входном
`arxiv.org/abs/<id>` или `/pdf/<id>` скилл мог бы сам попробовать `/html/<id>`, а при 404 —
вернуть подсказку «HTML нет, бери PDF» вместо того, чтобы вызывающий переписывал URL руками.

### Дефект №4 — полнота X-тредов варьируется. **Серьёзность: НИЗКАЯ (для осведомлённости)**
Фетч X теперь работает (большой плюс), но многотвитные треды берутся не целиком: Bons → 26 КБ
(вся ветка), а промо-тред espresso → 1.2 КБ (только головной твит; подгружаемые ответы не попали).
Реплаи «ниже сгиба» в длинных тредах могут теряться.

## 4. Постановка на доработку

- **R-7 (ВЫСОКИЙ) — гард на пустое извлечение + путь для Wikipedia/MediaWiki.**
  (a) Если извлечённое тело почти пустое (< N символов) при крупном исходном HTML — это НЕ exit 0:
  эмитить типизированный `EmptyExtraction` (новый тип/код в `--json-errors`), чтобы немые пустышки
  всплывали. (b) Авто-фолбэк для `wikipedia.org` через REST `page/html`-эндпоинт ИЛИ вторичный
  readability-экстрактор, когда trafilatura вернул пусто.
  *Критерий приёмки:* 3 wiki-URL из этого батча дают непустой Markdown (или явный `EmptyExtraction`),
  ни одного «exit 0 + пустое тело».
- **R-8 (НИЗКИЙ)** — починить stderr-строку (`html2pdf:` → корректный префикс; не обрезать).
- **R-9 (НИЗКИЙ)** — `arxiv.org/abs|pdf/<id>` → авто-попытка `/html/<id>` перед отказом; при 404 — подсказка про PDF.
