# marp-slide renderer — operator notes

Fully local (no global installs). All runtime dependencies live under `scripts/.venv/` and `scripts/node_modules/`.

## Quick start

    bash scripts/install.sh        # one-time setup; creates .venv + node_modules; idempotent
    scripts/render deck.md         # → deck.pptx with mermaid pre-rendered

`scripts/render` is a thin bash wrapper that invokes `render.py` through the local venv Python (`scripts/.venv/bin/python`). You can also call the Python directly:

    scripts/.venv/bin/python scripts/render.py deck.md --format pdf

## File layout

Committed (kept in the skill):

    scripts/
      .gitignore           # ignores .venv, node_modules, package-lock.json
      package.json         # pins @marp-team/marp-cli + @mermaid-js/mermaid-cli
      render.py            # main renderer
      render               # bash wrapper around .venv/bin/python render.py
      install.sh           # creates .venv + npm install (local) + smoke test
      README.md            # this file

Generated by `install.sh` (ignored):

    scripts/
      .venv/               # Python venv — no pip deps (stdlib only)
      node_modules/        # marp-cli + mermaid-cli + Puppeteer Chromium (~200 MB)
      package-lock.json    # (regenerated each install)

Per-deck artifacts (next to the input `.md`):

    deck.md
    deck_assets/
      diagram-<sha1>.mmd   # mermaid source (one per unique diagram)
      diagram-<sha1>.svg   # rendered SVG (served to marp)
    deck.pptx              # final output

The `_assets/` directory is safe to commit or `.gitignore` — project-specific choice.
SHA1 cache means re-running the same source skips `mmdc` calls.

## CLI flags

| Flag | Meaning |
|------|---------|
| `--format {pptx,pdf,html,png,jpeg}` | Output format. Default: `pptx`. |
| `--output PATH` | Explicit output path. Default: `<input_stem>.<format>`. |
| `--no-mermaid` | Skip mermaid preprocessing (faster; diagrams remain as code blocks). |
| `--strict-mermaid` | Fail if mermaid blocks exist but cannot be rendered. Default: warn and continue. |
| `--theme NAME` | Override the theme from frontmatter (name or CSS path). |
| `--mermaid-config PATH` | Pass `-c PATH` to mmdc. `scripts/mermaid-config.json` is auto-loaded when present (see Cyrillic/CJK section below). |

Exit codes:

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Bad arguments or missing input |
| 2 | `marp` CLI not available (run `install.sh`) |
| 3 | `marp` rendering failed or timed out (5 min per invocation) |
| 4 | Mermaid preprocessing failed / timed out AND `--strict-mermaid` was set |

## Security

`render.py` always invokes marp with `--allow-local-files`, which lets marp embed any file the current user can read (e.g. `![bg](/etc/passwd)` in a `.md` is read and embedded). **Render only decks you trust** — do not run this on `.md` files from untrusted sources.

## CI / sandboxed environments

`marp-cli` and `mermaid-cli` use Puppeteer, which downloads Chromium on first `npm install`. On restricted networks set these env vars **before** running `install.sh`:

    export PUPPETEER_SKIP_DOWNLOAD=true
    export PUPPETEER_EXECUTABLE_PATH=/path/to/chromium

Confirmed working with system Chrome at `/opt/google/chrome/chrome` (Linux) and `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome` (macOS).

## Cyrillic / CJK / RTL fonts in mermaid diagrams

`mmdc`'s default font set may not contain non-Latin glyphs — diagrams can render as boxes. Fix via a mermaid config JSON, applied in one of two ways:

**Option A (auto-load).** Drop a file named `mermaid-config.json` inside `scripts/`; `render.py` picks it up automatically on every run:

    cat > scripts/mermaid-config.json <<'EOF'
    { "theme": "default", "themeVariables": { "fontFamily": "Arial, sans-serif" } }
    EOF

**Option B (per-run).** Pass `--mermaid-config PATH` explicitly:

    scripts/render deck.md --mermaid-config /path/to/cfg.json

Either option is folded into the SHA1 cache key, so cached SVGs are invalidated automatically when the config file content changes. `mmdc --version` is included in the same key so toolchain upgrades also bust the cache.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| "marp not available" / "mmdc not found" | Run `bash scripts/install.sh`. |
| `install.sh` fails on `npm install` (Chromium download) | Set `PUPPETEER_SKIP_DOWNLOAD=true` + `PUPPETEER_EXECUTABLE_PATH`. |
| "local venv not found" from `scripts/render` | Run `bash scripts/install.sh` — `.venv/` was not created. |
| Diagrams look wrong after editing source | Delete the matching `_assets/diagram-*.svg` (or the whole `_assets/` dir) to bust the SHA1 cache. |
| Transparent diagram background clashes with dark themes | Edit `render.py`, change `-b transparent` to `-b white` or `-b "#0d1117"`. |
| Cyrillic text in mermaid renders as boxes | Add the mermaid config from the section above. |

## Validation

`install.sh` runs an inline smoke test (1-block mindmap → PDF). It now asserts three things: PDF non-empty, ≥1 SVG cached under `smoke_assets/`, and PDF size > 20 KB (catches the "SVG-reference broken → non-empty PDF without diagram" regression class).

For per-PR validation, add to CI:

    bash scripts/install.sh
    scripts/render examples/fixture-mermaid-minimal.md --output /tmp/minimal.pptx
    [ -s /tmp/minimal.pptx ] || exit 1
    scripts/render examples/fixture-mermaid-full-deck.md --output /tmp/full.pptx
    [ -s /tmp/full.pptx ] || exit 1
    [ "$(find examples/fixture-mermaid-full-deck_assets -name '*.svg' 2>/dev/null | wc -l)" -gt 0 ] \
        || { echo "no SVG cache — mmdc likely failed"; exit 1; }

    # Multi-block regex / dedup regression guard:
    scripts/render examples/fixture-mermaid-multi.md --output /tmp/multi.pptx
    [ -s /tmp/multi.pptx ] || exit 1
    # Fixture has 3 blocks where #1 and #3 are identical — expect exactly 2 unique SVGs.
    COUNT=$(find examples/fixture-mermaid-multi_assets -name 'diagram-*.svg' 2>/dev/null | wc -l | tr -d ' ')
    [ "$COUNT" -eq 2 ] || { echo "expected 2 SVGs for multi fixture, got $COUNT"; exit 1; }
