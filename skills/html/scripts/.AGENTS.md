# html/scripts — agent notes

`html` (formerly `html2md`) is the repo's first **TWO-master** skill (CLAUDE.md §2). Proprietary,
All Rights Reserved (embeds proprietary docx/pdf code — see ../NOTICE).

**Naming caveat (post-rename):** the skill + user-facing launcher is `html` — the launcher is
the **extensionless** `scripts/html` (NOT `html.py`: `scripts/` is `sys.path[0]`, so a `html.py`
— or renaming the package dir to `html` — would shadow the stdlib `html` module that `acquire.py`
and the pdf-mastered `web_clean/preprocess.py` import). The **internal Python package keeps its
`html2md` name** for the same reason; so `from html2md import …`, `import html2md`, the
`html2md/` package dir, `discover -s html2md/tests`, and the docx-mastered `html2md_core.js`
replica all retain the `html2md` token by design.

stay out of `sys.modules` after importing `web_clean.archives` + `.reader_mode`.

## html-owned files

- `html` — the extensionless launcher (FC-5 shim): venv bootstrap (`reexec_into_venv`) +
  re-exports from the `html2md` package. (NOT `html.py` — see the naming caveat above.)
  **Encapsulation:** the `__main__` block calls `cli._load_skill_env()` BEFORE `main()` —
  auto-loads `<skill>/.env` into the env so any caller (direct or via the
  `~/.claude/skills/html` symlink) gets the skill's config transparently, no machine-global
  pollution. Process env wins; opt-out `HTML_NO_DOTENV=1` (set by `test_e2e.sh` + the
  shim-subprocess tests for determinism); 0600/symlink-guarded. Loaded ONLY at the shim
  entry (importing the package — tests — never triggers it).
- `html2md.py` — the combined end-to-end command (`fetch` → `md` → delete the intermediate
  HTML), built from the two `html` primitives; delegates to the `html` launcher via `runpy`.
- `html2md/cli.py` — argparse surface (ARCH §5.1), path/URL resolution, exit-map,
  `_errors` envelope routing. `main`/`convert` wired in 022-05. **TASK 023:** new flags
  (`--engine …|remote`, `--no-remote`, `--remote-format`, `--target-selector`, `--search`,
  `--max-results`); `_validate_usage` (post-parse usage → `Usage`/exit 2); `_resolve_search_paths`;
  `_convert_one` (the single shared convert path — also the trust-markdown `content_kind=="markdown"`
  bypass) + `_convert_search` (per-result loop; search emits ONE note each). **TASK 024:** chrome-auth
  flags (`--chrome-storage-state`/`--chrome-cookies-file`/`--chrome-user-data-dir` mutually-exclusive,
  `--chrome-scroll`/`--chrome-scroll-passes`); `_validate_usage` env-fallback + **auth⇒engine chrome**
  + missing-file `BadInput` + **`--chrome-* ⊥ --search`** (no session fan-out); **`login` verb-intercept**
  in `main` (before the flat parser) → `_login_main`.
- `html2md/acquire.py` — FC-1: input dispatch + fetch. Offline 022-02; URL 022-06.
  **TASK 023 = the resilient fetch ladder (all html-owned, NOT gated):**
  `_acquire_url` is a **tier ladder** (`_url_tiers`: auto local-first `lite→chrome→remote`;
  `jina`/`remote` remote-first + local fallback; `lite`/`chrome` single tier; `allow_chrome=False`
  drops chrome for attacker-influenceable search results). Tiers: `_tier_lite`/`_tier_chrome`/
  `_tier_remote`. Remote tier `_fetch_remote_html` = **vendor-agnostic** provider loop
  (`_remote_providers`/`_build_reader_request`; `jina` default, `HTML_READER_URL`/`_PROVIDERS`
  for self-hosted). Classification: `_TierUnavailable` (escalate) vs terminal `FetchFailed`;
  one `FetchFailed(kind=all_engines_failed, details.tried=[…])` (URL-free trace) when exhausted.
  Privacy: target public-IP gate + `--no-remote` BEFORE any remote request; `_assert_safe_target`
  (CR/LF guard). **SSRF gate (`_is_ssrf_blocked`/`_host_is_public`/`_assert_public_http`):** keeps
  Python's maintained `ip.is_private` (+ loopback/link-local/reserved/multicast/unspecified) and
  SUBTRACTS an **explicit** carve-out — `HTML_SSRF_ALLOW_NETS`, **no code default**: unset OR
  `""` → NO carve-out (strict, fail-safe); a CIDR list → exactly those ranges. The shipped
  `.env.example` sets `198.18.0.0/15` (RFC 2544; `*.eth.limo`-style local-resolver mappings) so the
  auto-loaded `.env` re-allows it. IPv4-mapped/translated IPv6 unwrapped before the family-matched
  check; carve-out read lazily (the shim loads `.env` after import). Search:
  `_search_providers`/`run_search` (s.jina.ai default; per-result via the ladder). Trust-markdown:
  `_split_remote_markdown` + `_LOOKS_HTML` fallback.
  Heavy deps (httpx/trafilatura/playwright) imported lazily INSIDE functions only.
  **TASK 024 = authenticated Chrome (all html-owned, NOT gated):** `_fetch_chrome_html(url, opts)`
  is **SSRF-gated** — `_assert_public_http` before goto, `_install_chrome_guards` (context-level
  `route` aborts non-public requests, fail-closed, per-host getaddrinfo cache) installed BEFORE
  goto, off-target-public-redirect refusal (`_registrable` eTLD+1), `--max-bytes` parity on the
  rendered body, stale-session `is_login_wall`→`auth_required`. Auth context from
  `_chrome_auth.resolve_context_kwargs` (storage_state / cookies via `_cookies` / persistent
  profile). `_chrome_scroll` (bounded by passes + `_CHROME_SCROLL_BUDGET_S`). `_login_render`
  (headful mint, `umask 0o077` → 0600). `_import_sync_playwright` is the test seam.
  **TASK 025 (LIGHT):** both mint + render launch via `_launch_chrome_browser` /
  `_launch_chrome_persistent` — prefer the user's **real system Chrome** (`channel="chrome"`) with
  bundled-Chromium fallback (R10), pass `_CHROME_LAUNCH_ARGS`
  (`--disable-blink-features=AutomationControlled`), and `add_init_script(_WEBDRIVER_MASK_JS)` to
  mask `navigator.webdriver`. Fixes native logins (X email/password) being flagged as bots; Google
  OAuth may still block by design → cookie-export is the sanctioned path (KNOWN_ISSUES HTML2MD-10).
  Perf note: on a Chrome-less server (e.g. bundled-only Hermes) the `channel="chrome"` probe runs +
  fails fast per render before the bundled fallback — a cheap synchronous binary-resolve, no browser
  process starts on the failed attempt.

- `html2md/clean.py` — FC-2: calls `web_clean` (022-04). Never edits the replicas.
- `html_convert.js` — **html-owned** (NOT gated) turndown wrapper over the
  docx-mastered `html2md_core.js`; adds ARIA-role-table→GFM + chrome-button strip
  (web-page patterns docx doesn't have). The Node bridge spawns THIS file.
  **Math:** `htmlMath` rule converts Pandoc/MathJax `class="math inline|display"` spans
  (must carry inline|display, not bare `math`) → Obsidian `$…$`/`$$…$$` with RAW TeX (no
  markdown-escaping). **Images:** `htmlImage` keeps CONTENT-sized `data:` images
  (≥`DATA_URI_MIN_LEN`=1024 chars, whitespace-collapsed), drops tiny icon blobs.
- `html2md/md_clean.py` — **html-owned** post-turndown Markdown tidy: merge empty
  ATX headings (GitBook/Mintlify/Fern split headings), drop standalone chrome lines
  (Copy / Search… / Ask AI / feedback / AI-widget), drop **leaked widget markup**
  (`_is_attr_soup`: X-style button soup where a `[&>svg]:` Tailwind class's `>` breaks the tag,
  spilling `aria-*=/type="button"/data-state=…>` as text — gated on a widget attr + ≥2 attr
  tokens, fence-skipped), collapse blank runs. Code fences are inviolable.
  **Math normalization (`_normalize_math`):** converts surviving `\(…\)`/`\[…\]` (incl.
  turndown-double-escaped `\\(…\\)`) → `$…$`/`$$…$$`, mainly for the remote-reader (jina)
  trust-markdown path (turndown's own math is already `$`-delimited by the DOM rule). Code
  spans/fences protected. **Bracket forms gate on `_looks_like_math`** — turndown escapes
  plain `[recipient]`/`[1]` to `\[…\]`, so an unconditional convert would turn prose/citations
  into display math (vdd-multi regression). `_MD_UNESCAPE` does NOT touch `\\` (LaTeX
  line-break). `_MATH_DELIM_CAP` guards the O(n²) scan on pathological input.
