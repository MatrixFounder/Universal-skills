# Directory: skills/xlsx/scripts/

## Purpose

Single-file Python CLIs for the xlsx skill. Every script in this
directory takes a `.xlsx`/`.xlsm` path on argv and produces a
deterministic side-effect (a new file, a recalculated workbook, a
validation report, or — new for xlsx-6 — an inserted comment). All
scripts share three cross-skill contracts:

- **cross-3** (encryption): reader scripts call `office._encryption.assert_not_encrypted` to fail fast on legacy CFB / encrypted inputs (exit 3).
- **cross-4** (macros): reader/writer scripts emit a stderr warning via `office._macros.warn_if_macros_will_be_dropped` when `.xlsm → .xlsx` would silently strip VBA.
- **cross-5** (json-errors): every script exposes `--json-errors` to emit failures as a single-line JSON envelope (`{v, error, code, type?, details?}`) on stderr — see `_errors.py`.
- **cross-7-H1** (writer scripts): refuse same-path INPUT/OUTPUT (incl. via symlinks resolved through `Path.resolve()`) with exit 6 `SelfOverwriteRefused`.

The `office/`, `_errors.py`, `_soffice.py`, and `office_passwd.py`
modules are byte-identical to the docx master per CLAUDE.md §2 — do
NOT edit them here.

**Purpose:** CSV/TSV → styled `.xlsx` (bold header, frozen first row, auto-filter, leading-zero preservation).

### json2xlsx.py *(xlsx-2 — JSON → styled .xlsx, ✅ DONE)*

**Purpose:** Convert JSON / JSONL input (file path or stdin `-`) to a styled `.xlsx` workbook. The JSON parallel to `csv2xlsx.py`. Output styling mirrors csv2xlsx 1:1 (bold header, freeze, auto-filter, column widths). Cross-cutting parity: cross-5 `--json-errors` envelope, cross-7 H1 same-path guard (exit 6), stdin `-`. Public helper `convert_json_to_xlsx(input_path, output_path, **kwargs)` lives in `json2xlsx/__init__.py` (single source of truth).

Round-trip contract with future `xlsx2json.py` (xlsx-8): [`skills/xlsx/references/json-shapes.md`](../references/json-shapes.md).

#### xlsx_read/_merges.py — F3 *(✅ LIVE since 009-04)*
`parse_merges(ws)` returns a `MergeMap = dict[(row,col), (row,col)]`
of anchor→bottom-right (1-based, inclusive) from
`ws.merged_cells.ranges`. `apply_merge_policy(rows, merges, policy, *,
top_row, left_col) -> list[list]` is a **pure** function (never
mutates input); short-circuits with full inner-row copy when
`merges == {}` (vdd-multi P-M1 + L-N1 fix). Policies: `"anchor-
only"` (only anchor carries value); `"fill"` (broadcast); `"blank"`
(semantic alias for v1). `_overlapping_merges_check(ranges)` does
explicit O(n²) pairwise box-intersection (M8/D4 fix: openpyxl 3.1.5
silently accepts overlapping merges; library must detect explicitly).

#### xlsx_read/_tables.py — F4 *(✅ LIVE since 009-05)*
`detect_tables(wb, sheet_name, *, mode, gap_rows=2, gap_cols=1)`
runs the 3-tier detector:
- **Tier-1** `_listobjects_for_sheet`: iterates `ws.tables.keys()`
  (openpyxl 3.1 quirk: `TableList.items()` returns `(name, ref_str)`,
  not `(name, Table)`) and captures `headerRowCount`.
- **Tier-2** `_named_ranges_for_sheet`: iterates `ws.defined_names`
  (per-sheet `DefinedNameDict`; workbook-scope on `wb.defined_names`
  is ignored by construction — honest-scope D8). Multi-destination
  names get `#1`/`#2` disambiguation suffixes (vdd-multi L-M3 fix).
- **Tier-3** `_gap_detect`: uses `_resolve_dim_bbox` (parses
  `ws.dimensions` via `range_boundaries`, falls back to `(max_row,
  max_column)`); enforces `_GAP_DETECT_MAX_CELLS = 1_000_000` cap
  before allocating the occupancy grid (vdd-multi P-C1 fix —
  defends against malformed `<dimension ref="A1:XFD1048576"/>`);
  streams via `ws.iter_rows(values_only=True)` (vdd-multi P-C2 fix
  for `read_only=True` compatibility); splits via `_split_on_gap` +
  `_tight_bbox` + `_build_claimed_mask`. `_has_overlap` drops
  Tier-2 ranges intersecting Tier-1.

`mode="whole"` returns a single region spanning `ws.dimensions`;
`mode="tables-only"` skips Tier-3. Invalid mode raises `ValueError`.

#### xlsx_read/_headers.py — F5 *(✅ LIVE since 009-06)*
`detect_header_band(ws, region, hint)`:
- Integer `hint ≥ 0` → returns verbatim.
- `hint == "auto"`: tracks the maximum `merge.max_row` of any
  column-spanning merge **anchored** in the region's top rows
  (`merge.min_row >= region.top_row`, vdd-multi L-H1 fix — eliminates
  phantom-header from super-merges above tightened regions); header
  band runs through `max_merge_row + 1` (banner+sublabels heuristic),
  clamped to `region.bottom_row`. Default single-row if no merges.

`flatten_headers(rows, header_rows, separator=" › ")` (U+203A) with
sticky-fill-left + consecutive-repeat dedup; returns `(keys,
warnings)`.

`synthetic_headers(width)` is the **final** form (already from
009-01) — emits `col_1..col_N`; raises `ValueError` on negative width.

`_ambiguous_boundary_check(merges, region, header_rows)` returns a
warning string if any merge anchored in `[region.left_col,
region.right_col]` straddles the header/body cut.

#### xlsx_read/_values.py — F6 *(✅ LIVE since 009-07)*
