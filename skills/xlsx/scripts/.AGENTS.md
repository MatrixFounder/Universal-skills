# Directory: skills/xlsx/scripts/

## Purpose

Single-file Python CLIs for the xlsx skill. Every script in this
directory takes a `.xlsx`/`.xlsm` path on argv and produces a
deterministic side-effect (a new file, a recalculated workbook, a
validation report, or ŌĆö new for xlsx-6 ŌĆö an inserted comment). All
scripts share three cross-skill contracts:

- **cross-3** (encryption): reader scripts call `office._encryption.assert_not_encrypted` to fail fast on legacy CFB / encrypted inputs (exit 3).
- **cross-4** (macros): reader/writer scripts emit a stderr warning via `office._macros.warn_if_macros_will_be_dropped` when `.xlsm ŌåÆ .xlsx` would silently strip VBA.
- **cross-5** (json-errors): every script exposes `--json-errors` to emit failures as a single-line JSON envelope (`{v, error, code, type?, details?}`) on stderr ŌĆö see `_errors.py`.
- **cross-7-H1** (writer scripts): refuse same-path INPUT/OUTPUT (incl. via symlinks resolved through `Path.resolve()`) with exit 6 `SelfOverwriteRefused`.

The `office/`, `_errors.py`, `_soffice.py`, and `office_passwd.py`
modules are byte-identical to the docx master per CLAUDE.md ┬¦2 ŌĆö do
NOT edit them here.

### json2xlsx.py *(xlsx-2 ŌĆö JSON ŌåÆ styled .xlsx, Ō£ģ DONE)*

#### xlsx2md/emit_hybrid.py *(F3 ŌĆö format selector, LIVE since 012-06)*

#### xlsx2md/emit_html.py *(F5 ŌĆö HTML emitter, LIVE since 012-05)*

#### xlsx2md/headers.py *(F7 ŌĆö multi-row header reconstruction, LIVE since 012-05)*

Pure emit-side reconstruction of multi-row `<thead>` from flat ` ŌĆ║ `-joined `TableData.headers`. Separator is ` ŌĆ║ ` (U+203A with surrounding spaces, D6 lock).

**Public API:**
- `validate_header_depth_uniformity(headers) -> int` ŌĆö counts ` ŌĆ║ ` separators per header; if not uniform, raises `InconsistentHeaderDepth` (CODE=2, D-A11 defensive check). Returns depth N (separators + 1). First raise-site for `InconsistentHeaderDepth`.

## xlsx2md (xlsx-9 read-back to Markdown)

**Package:** `scripts/xlsx2md/`
**Shim:** `scripts/xlsx2md.py` (Ōēż 60 LOC).

### Closed-API consumption pattern

The package is the **second consumer** (after xlsx-8 / xlsx2csv2json)
of the xlsx-10.A `xlsx_read/` closed-API contract (ARCH D-A5).
Imports come exclusively from `xlsx_read.<public>`; the ruff
banned-api rule in `scripts/pyproject.toml` blocks any
`xlsx_read._*` import.

### Format selector (hybrid) decision rules

Per-table format selection in hybrid mode (default) ŌĆö first match wins:

| Rule | Condition | Format |
| --- | --- | --- |
| 1 | Body merges present | HTML |
| 2 | Multi-row header (` ŌĆ║ ` in any header) | HTML |
| 3 | `--include-formulas` AND Ōēź 1 formula cell | HTML |
| 4 | `headerRowCount=0` (synthetic headers, D13) | HTML |
| ŌĆö | None of the above | GFM |

`--format gfm` / `--format html` short-circuit the promotion
rules.

### Multi-row `<thead>` reconstruction (D-A11)

`xlsx_read` returns flat ` ŌĆ║ `-joined headers. xlsx-9
reconstructs the multi-row HTML `<thead>` emit-side:
`headers.split_headers_to_rows` splits each header into a
row-major matrix; `headers.compute_colspan_spans` groups
consecutive identical prefix paths into colspan values;
`headers.validate_header_depth_uniformity` raises
`InconsistentHeaderDepth` on non-uniform depth.

### `--no-table-autodetect` (D-A2 post-call filter)

The shim's `--no-table-autodetect` flag does NOT use a library
enum value (no `gap-only` mode in `xlsx_read.TableDetectMode`).
Instead, the dispatch calls `detect_tables(mode="auto")` then
filters with `r.source == "gap_detect"`. Mirror of xlsx-8 D-A2
pattern.

### `--memory-mode` (D-A14 inherited from xlsx-8a-11)

| `--memory-mode` | `read_only_mode` | Behaviour |
| --- | --- | --- |
| `auto` (default) | `None` | Library size-threshold (Ōēź 100 MiB ŌåÆ streaming) |
| `streaming` | `True` | Force read-only streaming; hyperlinks unreliable |
| `full` | `False` | Force in-memory; correct hyperlinks; ~5-10├Ś workbook size heap |

### `--hyperlink-scheme-allowlist` (D-A15 inherited from xlsx-8a-03)

Default `http,https,mailto`. Scheme filter applied to both GFM
`[text](url)` and HTML `<a href>` emissions. Blocked schemes
emit text-only + stderr `UserWarning`. Special values:
- `'*'` allows all schemes.
- `""` (empty) blocks all hyperlinks.
