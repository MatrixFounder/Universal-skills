# Directory: skills/xlsx/scripts/

## Purpose

Single-file Python CLIs for the xlsx skill. Every script in this
directory takes a `.xlsx`/`.xlsm` path on argv and produces a
deterministic side-effect (a new file, a recalculated workbook, a
validation report, or — new for xlsx-6 — an inserted comment). All
scripts share three cross-skill contracts:

- **cross-3** (encryption): reader scripts call `office._encryption.assert_not_encrypted` to fail fast on legacy CFB / encrypted inputs (exit 3).
- **cross-4** (macros): reader/writer scripts emit a stderr warning via `office._macros.warn_if_macros_will_be_dropped` when `.xlsm → .xlsx` would silently strip VBA.
- **cross-5** (json-errors): every script exposes `--json-errors` to emit failures as a single-line JSON envelope (`{v, error, code, type?, details?}`) on stderr — see `_errors.py`.
- **cross-7-H1** (writer scripts): refuse same-path INPUT/OUTPUT (incl. via symlinks resolved through `Path.resolve()`) with exit 6 `SelfOverwriteRefused`.

The `office/`, `_errors.py`, `_soffice.py`, and `office_passwd.py`
modules are byte-identical to the docx master per CLAUDE.md §2 — do
NOT edit them here.

## Files

### csv2xlsx.py
**Purpose:** CSV/TSV → styled `.xlsx` (bold header, frozen first row, auto-filter, leading-zero preservation).

### xlsx_recalc.py
**Purpose:** Force formula recalculation via headless LibreOffice; optional `--scan-errors`.

### xlsx_validate.py
**Purpose:** Fast scan for cached formula errors (`#REF!`, `#DIV/0!`, etc.) without recalc. `--fail-empty` catches the "formulas never recalculated" case.

### xlsx_add_chart.py
**Purpose:** Add a bar/line/pie chart over a cell range; chart stays editable in Excel/LibreOffice.

### xlsx_add_comment.py *(xlsx-6 v1 + Task-002 modular split)*

**Purpose:** Thin shim (≤ 200 LOC) — re-exports the 35-symbol
test-compat surface (TASK 002 §2.5) and delegates `main()` to
`xlsx_comment/cli.py`. The user-facing CLI surface is unchanged from
xlsx-6 v1; the split exists to make further development tractable
across 8 cohesive package modules (Task 002 / ARCH §3.2).

**Status:** v1 — full pipeline implemented (no stubs). Test surface:
86 unit tests (75 baseline + 11 Task-002 honest-scope/import-smoke)
+ 112 e2e tests + 5 canonical-XML goldens, all green.
`office/validate.py` clean on every produced workbook.
`validate_skill.py skills/xlsx` exit 0.

**Honest scope** (R9.a–R9.g) locked in
`tests/test_xlsx_add_comment.py::TestHonestScope`: no `parentId`
(reply threads), plain-text body only, default VML anchor only,
goldens are agent-output-only (Excel may mutate on save), UUIDv4
`<threadedComment id>` is non-deterministic by design, no
`--default-initials`, no `--unpacked-dir`. See
`references/comments-and-threads.md` §4 for the full list.
Task-002 honest-scope (no behaviour change beyond restructuring) is
locked by `tests/test_refactor_honest_scope.py`.

### xlsx_comment/ (subdirectory — Task-002 module split)

**Purpose:** Internal package backing `xlsx_add_comment.py`. 9 files
= 8 implementation modules + a near-empty `__init__.py` (Q4 closure).
Programmatic callers use the explicit submodule path
(`from xlsx_comment.cell_parser import …`); the shim is the user-facing
entry point.

#### xlsx_comment/__init__.py
Package marker. Near-empty per ARCH §8 Q4=A.

#### xlsx_comment/constants.py
XML namespaces (`SS_NS`, `R_NS`, `PR_NS`, `CT_NS`, `V_NS`, `O_NS`,
`X_NS`, `THREADED_NS`), OOXML rel-types, content-types,
`DEFAULT_VML_ANCHOR = "3, 15, 0, 5, 5, 31, 4, 8"` (R9.c lock),
`BATCH_MAX_BYTES = 8 * 1024 * 1024` (m-4 boundary).

#### xlsx_comment/exceptions.py
`_AppError` base + 14 typed leaves: `UsageError`, `SheetNotFound`,
`NoVisibleSheet`, `InvalidCellRef`, `MergedCellTarget`,
`EmptyCommentBody`, `InvalidBatchInput`, `BatchTooLarge`,
`MissingDefaultAuthor`, `DuplicateLegacyComment`,
`DuplicateThreadedComment` (M-2), `SelfOverwriteRefused`,
`OutputIntegrityFailure`, `MalformedVml`. Each carries `code` (exit
code) + `envelope_type` (JSON envelope `type` field) class attributes.

#### xlsx_comment/cell_parser.py
`parse_cell_syntax` (A1, Sheet2!B5, 'Q1 2026'!A1, 'Bob''s Sheet'!A1),
`resolve_sheet` (M2 first-VISIBLE / M3 case-sensitive with suggestion),
`_load_sheets_from_workbook` (private; reused by F6).

#### xlsx_comment/batch.py
`BatchRow` dataclass + `load_batch` (8 MiB cap, flat-array vs xlsx-7
envelope auto-detect, group-finding skip). Per Q5: BatchRow NOT
re-exported from the shim — programmatic callers import directly.

#### xlsx_comment/ooxml_editor.py *(largest module — single-file per Q1=A)*
The OOXML mutation core. Hosts `_VML_PARSER` (lxml hardened parser —
the security boundary against billion-laughs / XXE on tampered VML;
DO NOT mutate constructor). Scanners (`scan_idmap_used` list-aware
per M-1, `scan_spid_used`), part-counter (`next_part_counter`,
`_allocate_new_parts`), cell-ref helpers, target/path resolution,
rels + `[Content_Types].xml` patching, legacy comment writers
(`ensure_legacy_comments_part`, `add_legacy_comment`,
`ensure_vml_drawing`, `add_vml_shape`, `_xml_serialize`), threaded
comment writers (`ensure_threaded_comments_part`, `ensure_person_list`,
`add_person`, `add_threaded_comment`).

#### xlsx_comment/merge_dup.py
Merged-cell resolver (`resolve_merged_target` — fail-fast on non-anchor
unless `--allow-merged-target`) and duplicate-cell matrix
(`detect_existing_comment_state`, `_enforce_duplicate_matrix`;
ARCH §6.1).

#### xlsx_comment/cli_helpers.py
Pure utilities used by cli.py: `_initials_from_author`, `_resolve_date`
(Q5 closure), `_validate_args` (MX/DEP rules), `_assert_distinct_paths`
(cross-7 H1), `_content_types_path`, `_post_validate_enabled`
(env-truthy gate), `_post_pack_validate` (R8 / 2.08 guard via
office/validate.py subprocess).

#### xlsx_comment/cli.py
F1 (`build_parser` argparse) + F6 (`main`, `single_cell_main`,
`batch_main`) merged per Q2=A. `XLSX_ADD_COMMENT_POST_VALIDATE` env
var (truthy `1/true/yes/on`) gates the post-pack validate guard.
The unified `try / except _AppError / except EncryptedFileError`
handler routes typed exit codes through `_errors.report_error`.

### tests/
**Purpose:** E2E (`test_e2e.sh` — 112 passing tests, runs all xlsx skill
CLIs end-to-end including the xlsx-6 v1 surface + 5 canonical-XML
goldens) and Python unit tests:
- `test_xlsx_add_comment.py` — 75 tests, 0 skipped (xlsx-6 v1 unit surface).
- `test_refactor_honest_scope.py` — 4 tests (Task 002 R8 / R6.e
  regression locks: shim ≤ 200 LOC, 35-symbol re-export surface,
  `--help` description vs shim docstring drift, office/ 3-skill
  byte-equality).
- `test_xlsx_comment_imports.py` — 8 tests (Task 002 R7.d import-graph
  smoke: one per `xlsx_comment.*` module).
- `TestLegacyDrawingAnchor` (3 tests in `test_xlsx_add_comment.py`) —
  post-Task-002 hot-fix regression lock: `<legacyDrawing r:id=…/>`
  must be present in worksheet XML when a VML drawing is attached.
  Without it Excel sees the rel but does NOT render comment hover-
  bubbles (ECMA-376 Part 1 §18.3.1.27). Surfaced during real-world
  testing on `tmp/Анализ релизов.xlsx`.
- `TestRenderSmoke` (2 tests in `test_xlsx_add_comment.py`,
  `@skipIf(soffice missing)`) — complementary insurance: produced
  workbooks render via LibreOffice + Poppler without crashing the
  pipeline. Skip cleanly when `soffice` is not on PATH; run
  automatically when present. Catches malformed VML / broken rels /
  schema-out-of-order worksheets — but explicitly NOT pixel-level
  comment-marker visibility (out of scope; that requires a known-good
  PNG reference). Primary defence against the legacyDrawing class of
  bug remains `TestLegacyDrawingAnchor`.
- `office/tests/` — 43 tests (cross-skill office/ pack/unpack/validate).

Total xlsx-skill unit count: 92 + 43 = 135 (75 + 12 Task-002 + 3 hot-fix + 2 render-smoke + 43 office).
Render-smoke tests skip when LibreOffice is absent; run when present.
Helpers: `_golden_diff.py` (c14n + volatile-attribute mask),
`regenerate_synthetic_inputs.py` (rebuilds `golden/inputs/` from
synthetic specs).
