# wiki-ingest test suite

138 tests across 16 files. Invoked via the per-skill venv (per
[CLAUDE.md §1 Testing](../../../../CLAUDE.md#testing)):

```bash
cd skills/wiki-ingest/scripts
python3 -m venv .venv && source .venv/bin/activate
python -m unittest discover -s tests
```

## Layout

### Per-module unit tests
- `test__safety.py` — 13 tests: NFKC, traversal/control-char rejection,
  atomic-write under fsync failure (incl. M2-015-01 tmp-cleanup),
  read_text symlink refusal, JSON sanitisation.
- `test__markdown.py` — 14 tests: offset stability under mask, L-C3
  `---` non-boundary, mask-once invariant (10k headers <0.25s + ratio
  bound), inline-code/HTML-comment masking, abbreviation-aware
  `_first_sentence`, 16 KiB size cap.
- `test__frontmatter.py` — 14 tests: line-anchored `\n---` closer
  (L-C2 — 4-dash + trailing-text fixtures), `warnings=` out-parameter
  (L-M5), structural splice idempotency on collision-prone fixture,
  flow-list quote handling.
- `test__vault.py` — 10 tests: symlink-skipping `_walk_pages`, bucket
  layout, fenced-code-aware `tail_log` (L-M6), asset resolution,
  `ensure_schema` exit code.
- `test__classify.py` — 18 tests: binary-masquerade rejection (L-M8 —
  NUL + replacement-ratio fixtures), segment-aware hints (no
  `speculation/spec` substring), grouping pattern detection,
  `_UNGROUPED_SENTINEL` identity, 1 KiB streamed read.

### Per-command tests (`commands/`)
- `test_scan.py` (4), `test_init.py` (5), `test_upsert_page.py` (9),
  `test_update_index.py` (7), `test_append_log.py` (5),
  `test_log_event.py` (5), `test_register_summary.py` (7),
  `test_find_lint_reindex.py` (8), `test_classify_folder.py` (4).

Each per-command file locks: (a) `register()` attaches the right
subparser + sets `func=execute`, (b) happy-path `execute()`, (c) at
least one adversarial input where the command takes user data.

### Architecture invariant + R11 byte-identity + known-issues regression
- `test_architecture.py` (3) — ast-walks `wiki_ingest/` and asserts
  no `_*.py` imports `commands/*` and no `commands/<a>.py` imports
  `commands/<b>.py`. R7.4.
- `test_r11_byte_identity.py` (6) — runs `scan`/`lint`/`classify-folder`
  via `subprocess` against frozen fixtures with varied
  `PYTHONHASHSEED` (defeats CI envs that pin `PYTHONHASHSEED=0`).
  Stdout must byte-match `fixtures/expected/<cmd>.json`.
- `test_known_issues_resolved.py` (10) — regression tests for the
  15 KNOWN_ISSUES items resolved post-TASK-015 (S-L2, S-L3, L-H2,
  L-L3, L-L2, P-M3, P-M3-015-02, L-L9, P-M5, M2-015-01 — already
  covered in `test__safety` for the last one).

## Fixtures

- `fixtures/scan_vault/` — deterministic vault for `scan` R11 gate.
  **`log.md` is frozen** at `## [2024-01-01] init | fixture seed`.
  Do NOT run `append-log`/`log-event` against it — that would shift
  `last_log_entries` daily and break R11.
- `fixtures/lint_vault/` — exercises orphans, dangling links with
  anchors, contradiction blocks, case-fragmented concept aggregation.
- `fixtures/classify_folder/` — mixed-extension folder including a
  multi-candidate group (`01-transcript.txt` + `01-notes.md`) so
  `_pick_primary`'s mixed-hint branch is in the R11 lock.
- `fixtures/expected/{scan,lint,classify}.json` — captured stdout
  with `__VAULT_PATH__` placeholder for host portability.

## Adding a new fixture

1. Add the fixture under `fixtures/<scenario>/`.
2. Capture expected output once:
   `python3 wiki_ops.py <cmd> tests/fixtures/<scenario> | sed "s|$(pwd)/tests/fixtures/<scenario>|__VAULT_PATH__|g" > tests/fixtures/expected/<scenario>.json`.
3. Wire `test_<scenario>_byte_identity` into `test_r11_byte_identity.py`.
4. Commit fixture + expected in the same commit.

## What NOT to do

- Do NOT run `append-log` or `log-event` against any committed fixture
  vault. The `log.md` is intentionally frozen.
- Do NOT modify the expected JSON files unless a task description
  explicitly authorises a behavioural change. R11 = no drift.
- Do NOT relax the architecture-invariant test to make a forbidden
  import pass; fix the import.
