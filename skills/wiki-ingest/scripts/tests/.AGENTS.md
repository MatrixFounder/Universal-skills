# wiki-ingest test suite

Tests for the wiki-ingest skill — invoked via the per-skill venv:

```bash
cd skills/wiki-ingest/scripts
python3 -m venv .venv && source .venv/bin/activate
python -m unittest discover -s tests
```

## Layout

- `fixtures/scan_vault/` — deterministic vault for `scan` R11 gate.
  **`log.md` is frozen** (no `append-log` / `log-event` may be run
  against this fixture; that would shift `last_log_entries` daily).
- `fixtures/lint_vault/` — exercises orphans, dangling links, anchor
  links, contradiction blocks, and case-fragmented concept aggregation.
- `fixtures/classify_folder/` — mixed-extension folder for the
  `classify-folder` R11 gate.
- `fixtures/expected/{scan,lint,classify}.json` — captured stdout of
  each command against its fixture. **This is the byte-identity
  contract** that every TASK-015 bead must preserve.
- `test_r11_byte_identity.py` — runs the three commands via
  `subprocess` and asserts stdout byte-matches the expected file.
  Also runs each command 10× to assert determinism across runs.

## Adding a new fixture

If a bead requires a fixture not covered above:
1. Add the fixture under `tests/fixtures/<scenario>/`.
2. Capture the expected output once: `python3 wiki_ops.py <cmd>
   tests/fixtures/<scenario> > tests/fixtures/expected/<scenario>.json`.
3. Wire a new `test_<scenario>_byte_identity` into
   `test_r11_byte_identity.py`.
4. Commit fixture + expected together (same commit).

## What NOT to do

- Do NOT run `append-log` or `log-event` against any committed fixture
  vault. The `log.md` is intentionally frozen at `## [2024-01-01] init |
  fixture seed`.
- Do NOT modify the expected JSON files unless a bead's task description
  explicitly authorises a behavioural change. The whole point of R11 is
  that these files do NOT drift across the refactor.
