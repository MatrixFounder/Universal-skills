# wiki_ingest internal package

> **Status (as of bead 015-01):** only `_safety.py` is realised. The
> remaining modules (`_markdown.py`, `_frontmatter.py`, `_vault.py`,
> `_classify.py`, `commands/*`) land progressively in beads 015-02
> through 015-11. The layout below is the **target shape**; until each
> bead lands, the corresponding symbols still live in `wiki_ops.py`
> behind back-compat re-exports.

Internal package for the wiki-ingest skill. Layered model with a strict
one-way dependency rule:

```
F3 commands/*.py + _vault.py + _classify.py   ↑
F2 _markdown.py + _frontmatter.py             ↑
F1 _safety.py                                  (root)
```

- **F1 imports nothing from siblings.**
- **F2 imports only from `_safety` (F1).**
- **F3 imports from any of F1 / F2 / F3-siblings — but never from another command.**
- Future enforcement: `tests/test_architecture.py` (bead 015-06) walks the
  AST and fails the build on any forbidden import.

## What lives where

| Module           | Owns                                                                                                  |
|------------------|--------------------------------------------------------------------------------------------------------|
| `_safety.py`     | atomic I/O, NFKC slugify, `_safe_name`, `_safe_inline`, `_safe_for_json`, size-limit constants, `_collect_names`, `_LOG_FORBIDDEN_IN_DETAIL` |
| `_markdown.py`   | code-fence + inline-construct masking, section locators, wikilink extraction, `_first_sentence`        |
| `_frontmatter.py`| YAML split + flow-list parser + structural splice                                                      |
| `_vault.py`      | vault layout constants, `_walk_pages`, `load_vault_pages`, `tail_log`, `load_asset`                    |
| `_classify.py`   | folder-classification helpers (only used by `classify-folder`)                                          |
| `commands/<cmd>.py` | one CLI subcommand each; exposes `register(subparser)` + `execute(args)` only                       |

## What NOT to do

- Do NOT import from a `_*.py` module into a sibling on the same tier
  unless the dependency is acyclic (the rule above forbids cycles).
- Do NOT introduce a new helper module without first deciding which tier
  it lives on. If a function is used by ≥2 commands, it likely belongs
  in F1 or F2, not duplicated.
- Do NOT add runtime dependencies. wiki-ingest is pure stdlib.
- Do NOT touch `wiki_ops.py` from inside this package — the shim is the
  external contract; this package is its implementation.
