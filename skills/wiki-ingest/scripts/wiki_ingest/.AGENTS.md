# wiki_ingest internal package

Internal package for the wiki-ingest skill. Realised by
[TASK 015 — modular refactor](../../../../docs/tasks/task-015-wiki-ingest-modular-refactor.md)
(merged 2026-05-26). The 11 CLI subcommands live in `commands/*.py`,
the cross-cutting helpers in five `_*.py` modules. `wiki_ops.py` (69
LoC, sibling of this package) is now a thin argparse shim that
iterates `_COMMAND_MODULES` and calls each `register()`.

## Layered model — one-way dependency rule

```
F3 commands/*.py + _vault.py + _classify.py   ↑
F2 _markdown.py + _frontmatter.py             ↑
F1 _safety.py                                  (root)
```

- **F1 imports nothing from siblings.**
- **F2 imports only from `_safety` (F1).**
- **F3 imports from F1 / F2 / F3-siblings — but NEVER from another command.**
- **Enforced** by `tests/test_architecture.py` (ast-walk, ~30 LoC of real
  logic). Failing this test = the DAG is broken; fix the import, don't
  relax the test. **Known blind spot**: dynamic imports
  (`importlib.import_module`, `__import__`) are not detected — catch
  those in code review.

## Adding a new command

1. Create `commands/<your_cmd>.py` with `register` + `execute`.
2. Add it to `_COMMAND_MODULES` in `../wiki_ops.py` (2-line change: import + tuple entry).
3. Add `../tests/commands/test_<your_cmd>.py` with ≥1 register test + ≥1 execute happy-path test + ≥1 adversarial test where applicable.

## Where does a new helper go?

| Used by … | Goes in … |
|-----------|-----------|
| Exactly one command, no security/markdown content | Keep it command-local (private `_` prefix) |
| ≥2 commands, atomic I/O / NFKC / safe-name etc.  | `_safety.py` (F1) |
| ≥2 commands, markdown structure                   | `_markdown.py` (F2) |
| ≥2 commands, YAML frontmatter                     | `_frontmatter.py` (F2) |
| ≥2 commands, vault layout / walk / log            | `_vault.py` (F3-helper) |
| `classify-folder` only                             | `_classify.py` (F3-helper) |
| ≥2 commands cross-tier                            | usually means the abstraction is wrong — ask first |

## What NOT to do

- Do NOT import from a `_*.py` module into a sibling on the same tier
  unless the dependency is acyclic (the rule above forbids cycles).
- Do NOT introduce a new helper module without first deciding which tier
  it lives on. If a function is used by ≥2 commands, it likely belongs
  in F1 or F2, not duplicated.
- Do NOT add runtime dependencies. wiki-ingest is pure stdlib.
- Do NOT touch `wiki_ops.py` from inside this package — the shim is the
  external contract; this package is its implementation.
- Do NOT relax `tests/test_architecture.py` to make a forbidden import
  pass; fix the import.
