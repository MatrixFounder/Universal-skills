# Directory: skills/pptx/scripts/pptx2md/

## Purpose

pptx → Markdown converter (TASK 020). The read-back counterpart to `md2pptx.js`.
Pipeline over an explicit document model: `extract → (images, ocr) → emit`, wrapped
by a thin CLI. pptx-specific — **NOT** cross-skill replicated (mirrors `xlsx2md/`).
Consumes `_errors.py` + `office/_encryption.py` by **read-only import** (so the
4-/3-skill `diff -q` matrices stay silent — ARCH §9).

Entry: `../pptx2md.py` (shim: `_venv_bootstrap` prelude + `sys.path` + re-export `main`).

## Files

### `__init__.py`
- Closed public surface: `main`, `convert`, `_AppError` + subclasses. Inserts
  `scripts/` onto `sys.path` so `_errors`/`office` resolve standalone.

### `exceptions.py`
- `_AppError(RuntimeError)` base with `CODE` + `error_type` + `details`.
- Subclasses: `SelfOverwriteRefused` (6), `OcrEngineUnavailable` (1),
  `LanguagePackMissing` (1), `BadInput` (1), `InternalError` (1).
  `EncryptedFileError` (exit 3) is imported from `office._encryption`, not redefined.

### `model.py`
- Frozen dataclasses (inner lists mutable): `Deck`, `Slide`, `Block` union
  (`Heading`, `Bullets`/`BulletItem`, `Table`, `ImageRef`, `Placeholder`),
  `MediaAsset`, `PlaceholderAsset`, `OcrResult`.

### `cli.py`
- `build_parser()` — full argparse surface (ARCH §5.1).
- `_resolve_paths()` — INPUT strict-resolve (→`BadInput`), self-overwrite guard
  (→`SelfOverwriteRefused` exit 6), output-parent auto-create.
- `_resolve_media_dir()` — `(media_dir, link_base)`; file mode relative to the
  `.md`, stdout mode relative to CWD (D-7 / MAJOR-4).
- `main()` — envelope routing (exit map: 0/1/2/3/6); InternalError→1 redacted (AR-3).
  **020-01: runs guards then returns `_STUB_SENTINEL=-999`; full pipeline in 020-04.**
- `convert()` — programmatic API (stub until 020-04).

### `extract.py` — **STUB until 020-02**
- `assert_openable(path)` (encryption pre-flight), `build_deck(prs, opts)`.

### `images.py`
- `safe_image_meta(shape)` — guarded `(blob, sha1, ext, content_type)|None` read
  (AR-1: `.ext`/`.content_type` raise on EMF/SVG via Pillow). Single owner;
  consumed by `extract` + `materialise`.
- `materialise(deck, media_dir, link_base, *, no_images)` — writes deduped blobs
  (sha1, first-occurrence-wins `slide{N}-img{M}.{ext}`) to the sidecar dir from
  `deck.blobs`; returns `{ImageRef: MediaAsset|PlaceholderAsset}`. `--no-images` →
  `{}`, no dir created. Lazy dir creation; POSIX link base (D-7).

### model.py note
- `Deck.blobs` (`sha1 → (bytes, content_type)`) is the raw-image side table:
  `extract` populates it while walking pictures, `materialise` consumes it. Keeps
  `ImageRef` a pure pointer.

### `ocr.py` (opt-in; Pillow/tesseract touched only inside functions; cli imports it lazily)
- `probe(langs)` — `shutil.which` + dual-stream `tesseract --list-langs` parse →
  `OcrEngineUnavailable` / `LanguagePackMissing` (exit 1), fail-loud before output.
- `ocr_asset(blob, langs, timeout)` — Pillow-normalise blob → temp PNG (mkstemp,
  unlinked) → `subprocess.run(["tesseract", png, "stdout", "-l", langs], ...)` argv
  list, no shell; per-image failure/timeout → warn + `""`. Reuses pdf_ocr engine +
  `eng+rus` conventions; NO ocrmypdf/ghostscript/pytesseract (D-5). `cli._build_ocr_text`
  resolves the blob from `deck.blobs[asset.sha1]`, dedups per unique MediaAsset
  (`--jobs` parallel), and emit places `<!-- ocr -->` blocks. Verified end-to-end on
  the image-only slides-5 deck (recovered eng+rus text).
  - **Decompression-bomb guard is THREAD-LOCAL** (vdd-multi D-9): rejects an
    over-`Image.MAX_IMAGE_PIXELS` image via a lazy `Image.open(...).size` check
    BEFORE the full-decode `.save()` — NOT `warnings.catch_warnings()` (process-global,
    races under `--jobs>1`). Locked by an un-mocked `--jobs 2` regression test.

### `emit.py`
- `render_deck(deck, assets, ocr_text, opts)` — pure generator → Markdown: `## Slide
  N`, title→`### `, bullets (2-space indent/level), GFM tables (padded to header
  width), `![alt](rel)` image links, `<!-- ocr -->` blockquote under images, `[kind]`
  placeholders, `> **Notes:**` blocks (suppressed by `--no-notes`).

### `cli.py` (020-04: pipeline wired)
- `convert(input_path, output_path, args)` — open_deck → build_deck → materialise →
  (lazy ocr if `--ocr`) → atomic write. `_write_output` = `.partial`→`os.replace`
  (unlink on failure). `_build_ocr_text` lazy-imports `ocr` (body in 020-05).
  Perf baseline: slodes-3 (82 sl/16MB/136 imgs) → ~0.23s, no OCR.

### `tests/`
- `_fixtures.py` — build-at-runtime synthetic decks (python-pptx).
- `test_surface.py` — 020-01 frozen-surface unit tests (tightened in 020-02..05).
