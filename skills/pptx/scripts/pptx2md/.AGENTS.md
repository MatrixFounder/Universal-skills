### `ocr.py` (opt-in; Pillow/tesseract touched only inside functions; cli imports it lazily)
- `probe(langs)` â€” `shutil.which` + dual-stream `tesseract --list-langs` parse â†’
  `OcrEngineUnavailable` / `LanguagePackMissing` (exit 1), fail-loud before output.
- `ocr_asset(blob, langs, timeout)` â€” Pillow-normalise blob â†’ temp PNG (mkstemp,
  unlinked) â†’ `subprocess.run(["tesseract", png, "stdout", "-l", langs], ...)` argv
  list, no shell; per-image failure/timeout â†’ warn + `""`. Reuses pdf_ocr engine +
  `eng+rus` conventions; NO ocrmypdf/ghostscript/pytesseract (D-5). `cli._build_ocr_text`
  resolves the blob from `deck.blobs[asset.sha1]`, dedups per unique MediaAsset
  (`--jobs` parallel), and emit places `<!-- ocr -->` blocks. Verified end-to-end on
  the image-only slides-5 deck (recovered eng+rus text).
  - **Decompression-bomb guard is THREAD-LOCAL** (vdd-multi D-9): rejects an
    over-`Image.MAX_IMAGE_PIXELS` image via a lazy `Image.open(...).size` check
    BEFORE the full-decode `.save()` â€” NOT `warnings.catch_warnings()` (process-global,
    races under `--jobs>1`). Locked by an un-mocked `--jobs 2` regression test.
  - **WMF/EMF â†’ inline PNG** (`images.rasterise_vector`, D-11): vectors are rendered to
    PNG via `_soffice.convert_to` (LibreOffice) and written as the *sidecar media file*
    so they display inline. `images.materialise` pre-renders all unique vectors up front
    (`_prerender_vectors`, parallel over `--jobs`, sha1-deduped) and swaps `deck.blobs` to
    the PNG, so a later `--ocr` OCRs the PNG with **no second soffice call**. `ocr_asset`
    calls the *same* `rasterise_vector` only as a defensive fallback (e.g. `--no-images`,
    where nothing was pre-rendered). Soft-optional + **fail-closed** via a **bare
    `except Exception`** (a `RuntimeWarning`-as-error from the soffice hardened-shim path
    must not escape â€” AR-1): absent/failed soffice â†’ original `.wmf`/`.emf` bytes are kept
    instead. PNG **bytes** aren't byte-reproducible (LibreOffice metadata) but filenames +
    the `.md` are. **Perf:** ~2â€“3 s LibreOffice startup *per unique vector*; default
    `--jobs 1` is serial â€” raise `--jobs` on vector-heavy decks, or `--no-images` to skip.
  - **`--ocr-denoise` (opt-in noise reduction, TASK 021 / D-12).** OFF by default â†’
    the path above is byte-identical (`tesseract â€¦ stdout`, no dedup). When ON, three
    **subtractive-of-noise-only** filters: (a) **size-gate** â€” `ocr_asset` skips a
    raster whose `min(w,h) < min_px` (default 48; vector-rasterised PNGs exempt);
    (b) **confidence-gate** â€” `ocr_asset` appends the `tsv` config (outputbase stays
    `stdout`) and pure helper `_filter_tsv(tsv, min_conf, min_words=2)` keeps words
    with `conf >= min_conf` (default 50), drops the image when **< 2** survive, and
    strips low-conf garble from kept blocks. **Gate on the COUNT of confident words,
    NOT the mean** â€” a mean gate dropped dense real screenshots whose UI chrome lowers
    the average (calibration finding, locked by dogfood + `test_ocr_denoise.py`).
    Malformed/headerless TSV â†’ `""`, never raises (AR-1). The dedup filter lives in
    `emit.render_deck` (below). All pptx-specific â€” no replicated file touched.
