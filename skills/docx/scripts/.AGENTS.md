## Script inventory

### `docx_replace.py` ‚Äî 500 LOC ‚Äî last modified: 2026-05-12 (task-007 [LIGHT] docx-6.7 `--scope` filter)

Surgical anchor-and-action editor for live `.docx` / `.docm` files.
Implements F1 (pre-flight: unpack, encryption-check, validate) and
F7 (CLI orchestration: argument parsing, dispatch, pack, post-validate).
Action logic lives in `_actions.py` (extracted per task 006-07a
Q-A1 LOC guardrail). Three mutually-exclusive actions: `--replace`,
`--insert-after`, `--delete-paragraph`. Anchor is always required
(`--anchor TEXT`). `--all` processes every match; default processes
only the first. `--unpacked-dir DIR` skips unpack/pack for library
(in-process) callers. `--json-errors` routes all failures through the
cross-5 JSON envelope (`{v:1, error, code, type, details}`).
Exit codes: 0/1/2/3/6/7. Honest-scope limits documented in `--help`
and enforced by R10.a‚ÄìR10.e + Q-U1 + A4 TOCTOU locks in
`tests/test_docx_replace.py::TestHonestScopeLocks`.

Tests: `tests/test_docx_replace.py` ‚Äî 62 unit tests, all live (0 skip),
26 + 48 = 74 total across both docx-6 unit files.

### `docx_anchor.py` ‚Äî 175 LOC ‚Äî last modified: 2026-05-12 (post-VDD-Multi Phase-3)

Anchor utility library shared by `docx_replace.py` and
`docx_add_comment.py`. Exports three public callables:

- `_find_paragraphs_containing_anchor(body_root, anchor, *, anchor_all)` ‚Äî
  walks `<w:body>` paragraphs, calls `_merge_adjacent_runs` on each,
  returns list of `(paragraph_element, run_element, char_offset)` tuples
  where the anchor text was found. Returns first match only unless
  `anchor_all=True`.
- `_merge_adjacent_runs(paragraph)` ‚Äî coalesces adjacent `<w:r>` elements
  with identical `<w:rPr>` (or both lacking `<w:rPr>`) into a single run,
  operating in-place. Required before anchor search to handle Word's
  split-run behaviour after autocorrect / style application.
- `_replace_in_run(run, anchor, replacement)` ‚Äî replaces the first
  occurrence of `anchor` in a `<w:t>` element's text; sets
  `xml:space="preserve"` when the replacement has leading or trailing
  whitespace.

Honest scope: anchor must fit within a single `<w:t>` after
`_merge_adjacent_runs`. Cross-run anchors spanning a format boundary
are not supported in v1.

Tests: `tests/test_docx_anchor.py` ‚Äî 26 unit tests, all live.

### `_actions.py` ‚Äî 431 LOC ‚Äî last modified: 2026-05-12 (task-007 [LIGHT] docx-6.7 `scope` param plumbing)

Action helpers for `docx_replace.py` (extracted at task 006-07a to
keep `docx_replace.py` within the 600 LOC budget per Q-A1 guardrail).
Module is docx-only (NOT under `office/`), so the cross-skill
replication boundary in `CLAUDE.md ¬ß2` is preserved.

Owns four functional blocks:

- **F2 part-walker** (`_iter_searchable_parts`) ‚Äî yields
  `(part_path, root_element)` for every searchable XML part in
  deterministic order: document ‚Üí headers (sorted) ‚Üí footers (sorted)
  ‚Üí footnotes ‚Üí endnotes. Primary source: `[Content_Types].xml`
  Override entries; filesystem glob is a fallback with a stderr
  warning. Implements R5 (part-walker enumeration).
- **F4** (`_do_replace`) ‚Äî walks parts, calls `_merge_adjacent_runs`
  then `_find_paragraphs_containing_anchor`, rewrites `<w:t>` text
  in-place. Implements R3.a‚ÄìR3.e.
- **F5** (`_do_insert_after`) ‚Äî locates the anchor paragraph and
  appends a new `<w:p>` after it. New paragraph is generated from a
  Markdown source via `md2docx.js` (subprocess). Implements R4.
- **F6** (`_do_delete_paragraph`) ‚Äî locates the anchor paragraph and
  removes it from `<w:body>`. Refuses to delete the last paragraph in
  `<w:body>` (`LastParagraphCannotBeDeleted`, exit 2). Implements R6.

### `_app_errors.py` ‚Äî 64 LOC ‚Äî last modified: 2026-05-12

Domain exception hierarchy for `docx_replace.py`. All exceptions
inherit from `DocxReplaceError` (itself from `Exception`). Enumerates:

- `AnchorNotFound` ‚Äî anchor TEXT not found in any searchable part.
- `LastParagraphCannotBeDeleted` ‚Äî `--delete-paragraph` refused because
  only one `<w:p>` remains in `<w:body>`.
- `Md2DocxNotAvailable` ‚Äî `md2docx.js` not found in PATH or scripts dir.
- `Md2DocxFailed` ‚Äî `md2docx.js` subprocess returned non-zero.
- `Md2DocxOutputInvalid` ‚Äî `md2docx.js` succeeded but output is not a
  valid ZIP / `.docx`.
- `EmptyInsertSource` ‚Äî `--insert-after -` (stdin) provided empty content.

## Other Python scripts in this directory

### `docx_add_comment.py` ‚Äî 1060 LOC ‚Äî last modified: 2026-05-12 (XXE hardening per VDD-Multi)

Adds a Word review comment anchored on a text substring. Wires
`<w:commentRangeStart>`/`<w:commentRangeEnd>`/`<w:commentReference>`.
Supports threaded replies (`--parent N`) and library mode
(`--unpacked-dir DIR`). See `SKILL.md ¬ß2` for full capability notes.

### `docx_fill_template.py` ‚Äî 235 LOC

Fills `{{placeholder}}` / `{{nested.key}}` markers in a `.docx`
template from a JSON payload. Run-merging pass prevents split-run
placeholders from silently missing. `--strict` fails on any unresolved
placeholder.

### `docx_accept_changes.py` ‚Äî 146 LOC

Drives LibreOffice's tracked-change accept dispatcher headlessly.
Requires `soffice` in PATH. Idempotent on already-accepted documents.

### `docx_merge.py` ‚Äî 827 LOC

Merges N `.docx` files into one: body content, styles, numbering,
media, relationships all relocated; `<w:bookmarkStart/End>` IDs bumped;
`<w:abstractNum>`/`<w:num>` shifted. See `SKILL.md ¬ß2` for full scope.
